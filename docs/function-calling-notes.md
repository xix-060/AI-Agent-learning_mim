**# Function Calling** 完整链路

**##** 什么是 **Function Calling**？

让 LLM 不只是"说话"，⽽是能"调⽤函数"。

LLM 本⾝不执⾏代码，它只是**\*\***输出要调⽤哪个函数、传什么参数**\*\***（JSON），

真正的执⾏在你的代码⾥。

**##** 完整流程（**6** 步）

1\. 用户输⼊："北京今天天⽓怎么样？"

2\. 你的代码把 \[用户问题 + 可⽤⼯具列表（JSON Schema）

] 发给 LLM

3\. LLM 返回：{"name": "get*\_weather", "arguments": {"city": "北京"}}*

*4. 你的代码解析这个 JSON，本地执⾏ get\_*weather("北京")

5\. 把函数执⾏结果发回给 LLM

6\. LLM 基于结果⽣成⾃然语⾔回复："北京今天晴，25°C"

**##** 关键点

\- LLM 不执⾏函数，只输出"调⽤意图"

\- 函数的执⾏在你的 Python 代码⾥

\- 需要⽤ JSON Schema 描述每个函数的签名（名字/描述/参数）

\- 这就是 Agent 的核⼼循环：感知 → 决策 → 执⾏ → 观察
