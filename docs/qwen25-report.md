## 1. 训练数据（第2节：预训练数据）

### 数据总量

| 版本 | 数据量 |
|------|--------|
| Qwen2 | 7T tokens |
| **Qwen2.5** | **18T tokens** |

数据量翻了近 2.6 倍。

### 四大改进

**（1）更好的数据过滤（Better Data Filtering）**

- 用 Qwen2-Instruct 模型作为数据质量过滤器
- 对训练样本进行多维度分析、评分和过滤
- 比上一代方法更精细：保留更多高质量数据，过滤更多低质量数据
- 支持多语言过滤

**（2）更好的数学和代码数据（Better Math & Code Data）**

- 将 Qwen2.5-Math 和 Qwen2.5-Coder 的训练数据整合进预训练
- 这些专用数据集是数学和代码任务达到 SOTA 的关键
- 让 Qwen2.5 天生就继承数学推理和代码生成能力

**（3）更好的合成数据（Better Synthetic Data）**

- 用 Qwen2-72B-Instruct 和 Qwen2-Math-72B-Instruct 生成高质量合成数据
- 主要覆盖数学、代码、知识领域
- 用通用奖励模型和 Qwen2-Math-RM-72B 对合成数据做二次过滤

**（4）更好的数据混合策略（Better Data Mixture）**

- 用 Qwen2-Instruct 对数据做领域分类和平衡
- 发现问题：电商、社交媒体、娱乐等领域在网页数据中占比过高（含大量重复/模板化/机器生成内容）
- 科技、学术等领域虽然质量高但占比过低
- 策略：对过度代表领域做降采样，对高价值领域做升采样 → 更均衡、信息更丰富的训练集

## 3. 架构特点（第3.1节）

### 核心架构 = Transformer Decoder + 五大改进

```
Qwen2.5 = Transformer Decoder
         + GQA（分组查询注意力）
         + SwiGLU（激活函数）
         + RoPE（旋转位置编码）
         + RMSNorm（归一化）
         + QKV Bias（注意力偏置）
```

### 3.1 GQA（分组查询注意力）

**对比 MHA vs GQA vs MQA**：

| | MHA（原始） | GQA（Qwen2.5） | MQA（极端） |
|---|---|---|---|
| Q 头数 | 所有头 | 所有头 | 所有头 |
| K/V 头数 | = Q头数 | 少于Q头数（共享） | = 1 |
| 72B模型 | 64/64 | **64/8** | 64/1 |
| KV Cache | 最大 | **减少87.5%** | 最小 |
| 推理速度 | 最慢 | **提升4~6倍** | 最快 |
| 性能 | 最高 | **接近MHA** | 略有损失 |

GQA 的操作步骤：
1. 把 Q 头分组（如72B模型：64个Q头分8组，每组8个Q头）
2. 每组共享1组 KV 头
3. 组内所有 Q 头用同一组 KV 计算注意力 → 大幅减少 KV Cache

### 3.2 SwiGLU 激活函数

$$\text{SwiGLU}(x) = \text{Swish}(xW_1) \odot (xV)$$

- Swish 函数：x · sigmoid(βx)，比 ReLU 更平滑
- 门控机制：一个分支控制"信息流量"，另一个分支决定"信息内容"
- FFN 维度从 4×d_model 缩减到 8/3×d_model，降低计算成本

### 3.3 RoPE（旋转位置编码）

- 不再给向量"加"位置信息，而是"旋转"向量
- Q 和 K 的点积天然包含相对位置信息
- Qwen2.5 将 RoPE 的 base frequency 从 1,000,000 提升到 **10,000,000**
- 更大的 base → 天生支持更长上下文
- 用 FP32 精度存储逆频率矩阵 → 保证长文本建模精度

### 3.4 RMSNorm + Pre-Normalization

- 用 RMSNorm 替代 LayerNorm → 不算均值，只算根均方 → 更快
- 使用 Pre-Norm（先归一化再做子层变换）→ 训练更稳定
- 没有偏移参数 β → 更少的参数量

### 3.5 QKV Bias

- 在注意力机制的 Q/K/V 投影中加入偏置（bias）
- 小模型中使用 QKV bias，大模型中部分使用
- 帮助小模型更快收敛

### 3.6 与 LLaMA 的差异

| 特性 | LLaMA | Qwen2.5 | 差异说明 |
|------|-------|---------|---------|
| 归一化 | RMSNorm | RMSNorm | 相同 |
| 位置编码 | RoPE | RoPE | Qwen base更大(10M vs 500K) |
| 激活函数 | SwiGLU | SwiGLU | 相同 |
| 注意力 | GQA(大模型) | **GQA(所有模型)** | Qwen全面采用GQA |
| QKV Bias | 无 | **有** | Qwen额外加了偏置 |
| 词表大小 | 32K | **151,643** | Qwen中文支持更好 |
| 上下文 | 4K~8K | **32K~128K** | Qwen支持更长上下文 |

### 3.7 分词器

- 使用 byte-level BPE（BBPE）编码
- 词表大小：**151,643** 个常规 token
- 控制token从3个增加到22个（新增2个工具调用token）
- 统一词表覆盖所有 Qwen2.5 模型
- 中文编码效率高

## 4. 后训练（第4节：SFT + RLHF）

### 4.1 SFT（监督微调）

- 使用超过 **100万条** 高质量样本
- 涵盖对话、指令跟随、代码、数学、多语言等多个领域
- 精心设计 prompt 格式和 response 质量

### 4.2 RLHF（多阶段强化学习）

Qwen2.5 实施了多阶段强化学习，而非简单的单步 RLHF：

**阶段概览**：
1. **离线 RL** → 用奖励模型对已有回复打分，构造偏好数据
2. **在线 RL** → 模型实时生成回复，奖励模型实时打分，动态优化策略
3. **多轮对话 RL** → 在多轮对话场景中做强化学习

**奖励模型**：
- 通用奖励模型：评估回复的整体质量
- 专用奖励模型（如 Math-RM）：针对特定领域评估

### 4.3 后训练效果

- 显著改善人类偏好对齐
- 长文本生成能力提升
- 结构化数据分析能力增强
- 指令跟随更精准
- 工具调用能力改善
