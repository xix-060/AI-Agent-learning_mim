# Attention Is All You Need 阅读笔记

## 1. 这篇论文解决什么问题？（用 1 句话回答）

**只用注意力机制（不用RNN也不用CNN），就能做序列到序列的任务（如机器翻译），而且效果更好、训练更快。**

> 补充背景：之前做翻译主要靠 RNN/LSTM，它们必须逐词处理，存在三个致命问题——慢（无法并行）、忘（长距离信息衰减）、难训练。Transformer 用"一眼看全句"的注意力机制彻底解决了这些问题。

## 2. Scaled Dot-Product Attention 公式

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

### Q 是什么？

**Query（查询矩阵）**——代表"我想找什么"。每个词生成一个查询向量，用来去问其他词："你跟我有关系吗？"

> 类比：你在图书馆搜索框里输入的关键词。

### K 是什么？

**Key（键矩阵）**——代表"我是什么标签"。每个词生成一个键向量，用来回答别的词的查询："我属于什么类型。"

> 类比：书架上每本书的分类标签。

### V 是什么？

**Value（值矩阵）**——代表"我的实际内容"。每个词生成一个值向量，存放真正要传递的信息。

> 类比：书里面的正文内容。

### 为什么要除以 √d_k？

当维度 d_k 很大时，Q 和 K 的点积结果数值会变得很大（方差随维度增长），导致 softmax 输出接近 0 或 1（极端分布），进入梯度极小区域 → **梯度消失**，训练几乎无法进行。

除以 √d_k 相当于把方差拉回到 1，让 softmax 输出分布更平滑，梯度更健康。

> 直觉：就像考试分数太集中时，调一下评分标准，让分数拉开差距，别全挤在 0 或 100。

### softmax 作用是什么？

**归一化为概率分布**——把任意一组实数变成 [0, 1] 之间的概率值，且总和为 1。这样每个词对其他词的"关注度"就是一个百分比，方便加权求和。

> softmax(x_i) = e^{x_i} / Σ e^{x_j}

## 3. Multi-Head Attention 的作用

### 为什么用多个头而不是一个？

**一个头只能学一种关注模式，多个头可以同时从不同角度理解同一句话。**

具体来说：

| 头 | 可能学到的关注模式 | 例子 |
|---|---|---|
| 头1 | 语法依赖 | "喝"关注"一杯"（动词→数量） |
| 头2 | 语义搭配 | "一杯"关注"咖啡"（量词→名词） |
| 头3 | 远距离指代 | 句首的"我"关注句尾的"咖啡" |
| 头4-8 | 其他模式 | 否定词、情感词、介词关系等 |

论文用 8 个头（h=8），每个头的维度是 d_k = d_model / h = 512 / 8 = 64。

**计算流程：**
1. 把 Q、K、V 各切 8 份
2. 每份独立做 Scaled Dot-Product Attention
3. 8 个结果 Concat（拼接）在一起
4. 过一个线性层 W^O 融合 → 最终输出

> 一句话：多头 = 多视角并行观察 → 拼接 → 融合，比单头能看到更丰富的信息。

## 4. Transformer 的三大组件

### 1. Embedding 层

把离散的词（符号）变成连续的向量（数字），让机器能计算。

- **Input Embedding**：把原文每个词变成 d_model=512 维的向量
- **Output Embedding**：把译文每个词变成同样维度的向量
- **位置编码**：加到 Embedding 上，给每个词注入顺序信息（因为 Transformer 本身没有顺序感）

> 公式：最终输入 = Embedding(x) + PositionalEncoding(pos)

### 2. Attention 层（Self-Attention + Cross-Attention）

**Self-Attention（自注意力）**：同一个序列内部的词互相关注。

- Encoder 里用的是 Self-Attention → 每个词看原文里其他所有词
- Decoder 里的 Masked Self-Attention → 只能看已经生成的词，不能偷看后面的

**Cross-Attention（交叉注意力）**：Decoder 去关注 Encoder 的输出。

- Q 来自 Decoder（"我在写什么，想参考原文哪部分？")
- K 和 V 来自 Encoder（"原文的每个词是什么标签、什么内容？")
- 这是两条流水线之间唯一的连接通道

### 3. Feed-Forward 层

每个位置独立过同样的两层全连接网络：

$$\text{FFN}(x) = \text{ReLU}(xW_1 + b_1)W_2 + b_2$$

- 第一层：把 512 维放大到 2048 维（展开、提取更多特征）
- 第二层：把 2048 维压缩回 512 维（浓缩、提炼精华）

> 这个层让每个词有机会"独立深度思考"，不依赖和其他词的关系。

**三个组件在每个 Encoder/Decoder 层里都出现，且每个组件后都有 Add & Norm（残差连接 + 层归一化）。**

## 5. Encoder 和 Decoder 的区别

| 特征 | Encoder（编码器） | Decoder（解码器） |
|---|---|---|
| **注意力方向** | **双向**——每个词可以看所有词 | **单向（因果 mask）**——只能看已生成的词 |
| **注意力种类** | 只有 Self-Attention | 有 Masked Self-Attention + Cross-Attention |
| **输入来源** | 原文序列 | 已生成的译文 + Encoder 输出 |
| **输出用途** | 生成对原文的理解表示 | 逐词生成译文 |
| **层数** | 6 层 | | 6 层 |
| **能否偷看未来** | 可以（理解整句） | **不行**（Mask 遮住后面的词） |

### Masked Attention 具体怎么做？

在算 softmax 之前，把未来位置的匹配分数设为 -∞（或一个非常大的负数），这样 softmax 之后那些位置的权重就变成 ≈ 0，相当于"遮住"了。

> 就像考试写作文，你只能看到已经写出来的部分，不能翻到后面看答案。

## 6. 位置编码（Positional Encoding）

### 为什么需要？

Transformer 没有 RNN 的天然顺序感——它同时处理所有词，不知道谁在前谁在后。

**没有位置编码的后果：** "狗咬人"和"人咬狗"对 Transformer 来说是一模一样的（词相同，顺序不同但它看不出来）。

所以必须额外注入顺序信息，让每个词知道自己坐在"第几个位置"。

### 用什么公式？

$$\text{PE}(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d}}\right)$$

$$\text{PE}(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$

- pos：词在序列中的位置（0, 1, 2, ...）
- i：向量中的维度编号（0, 1, 2, ...）
- d：模型维度（512）

### 这个公式的巧妙之处

1. **不同维度用不同频率** → 低维度变化快（区分相邻位置），高维度变化慢（区分远距离位置）
2. **sin/cos 组合** → 可以通过线性变换推导出相对位置关系（PE(pos+k) 可以用 PE(pos) 的线性组合表示）
3. **值域有界** → sin/cos 在 [-1, 1] 之间，不会把 Embedding 的数值范围搞乱
4. **不需要学习** → 固定公式，直接计算即可（论文也试了学习式位置编码，效果差不多，但固定式更简单且能处理超长序列）

> 直觉：就像给每个词发了一个独特的"身份证号"，低频维度是省份，高频维度是门牌号，组合起来既能定位绝对位置，又能算出相对距离。
