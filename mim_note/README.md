## 笔记结构说明（我的六段式框架）

每个知识点都按以下结构组织，这不是模板填充，是我思考一个技术问题时的真实路径：

```
1. 一句话破题    → 用一句话把这个概念的本质说透
2. 核心三问      → 面试官最可能追问的三个方向
3. 类比记忆法    → 用生活中的东西建立直觉锚点
4. 深度拆解      → 技术细节，该硬核的地方硬核
5. 面试话术      → 可以直接在面试场上说的话（第一人称）
6. 雷区雷达      → 别人踩过的坑，你别再踩
```

## 怎么用这套笔记

**面试前 3 天**：通读一遍，重点看「类比记忆法」和「面试话术」，建立直觉。

**面试前 1 天**：只看「一句话破题」和「雷区雷达」，快速过一遍知识地图。

**面试现场**：被问到时，先在脑子里调出「类比记忆法」建立框架，再用「深度拆解」里的细节填充，最后用「面试话术」的节奏说出来。

**面试后**：把被问住的内容补进对应章节，持续迭代。

---

## 关于风格

- **不说废话**：每个字都要有用，不说"这是一个很好的问题"
- **用比喻但不滥用**：比喻是为了建立直觉，不是为了显得有趣
- **第一人称**：面试话术部分用"我"来说，因为面试时你就是这么说的
- **有观点**：不只是罗列知识点，会说"我认为""实际项目里我倾向于"
- **承认不确定**：不确定的地方直接说不确定，比硬编强一百倍
